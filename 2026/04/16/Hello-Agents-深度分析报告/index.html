一、项目概述

1.1 项目定位

Hello-Agents 是 Datawhale 社区推出的系统性智能体学习教程，定位为”从零开始构建智能体”的实战指南。项目强调 AI 原生 Agent 的构建，而非简单的流程驱动型 Agent（如 Dify、Coze 等）。

1.2 核心价值主张

  • 完全免费开源
  • 理论与实战并重
  • 穿透框架表象，从第一性原理出发
  • 亲手构建属于自己的智能体框架

6.3 提供商自动检测逻辑

  1. 环境变量优先级检测

    • OPENAI_API_KEY → openai
    • DEEPSEEK_API_KEY → deepseek
    • DASHSCOPE_API_KEY → qwen
    • MODELSCOPE_API_KEY → modelscope
    • KIMI_API_KEY / MOONSHOT_API_KEY → kimi
    • ZHIPU_API_KEY / GLM_API_KEY → zhipu
    • OLLAMA_API_KEY / OLLAMA_HOST → ollama
    • VLLM_API_KEY / VLLM_HOST → vllm

  2. API 密钥格式检测

    • 以 ms- 开头 → modelscope
    • 以 sk- 开头且长度 > 50 → openai/deepseek/kimi
    • 包含 . → zhipu

  3. Base URL 检测

    • api.openai.com → openai
    • api.deepseek.com → deepseek
    • dashscope.aliyuncs.com → qwen
    • api-inference.modelscope.cn → modelscope
    • api.moonshot.cn → kimi
    • open.bigmodel.cn → zhipu
    • localhost:11434 → ollama
    • localhost:8000 → vllm

七、核心设计模式总结

7.1 架构设计模式

模式 应用场景 实现位置
抽象基类 Agent、Tool 统一接口 core/agent.py, tools/base.py
组合模式 MemoryManager 组合多种记忆类型 memory/manager.py
适配器模式 LLMAdapter 适配多种提供商 core/llm.py
单例模式 LLM 客户端全局复用 services/llm_service.py
工厂方法 动态创建 LLM 实例 llm.py build_llm()

7.2 设计亮点

  1. 统一的提供商抽象：支持多种 LLM 提供商，自动检测切换
  2. 智能自动检测：根据 API 密钥格式和 Base URL 自动识别提供商
  3. 流式响应优先：默认使用流式输出，提升用户体验
  4. 灵活的配置系统：支持 dataclass、Pydantic、环境变量多种配置方式
  5. 分层架构：基础设施层、记忆类型层、存储后端层、嵌入服务层，职责清晰
  6. 混合检索：语义向量 + 关键词 + 时间衰减 + 重要性权重
  7. 优雅降级：无向量库时回退到文本检索，无CLIP/CLAP时使用哈希向量
  8. Token预算管理：ContextBuilder 提供预算守护和压缩策略

8.3 三大 Agent 范式选择指南

范式 核心机制 优势 适用场景
ReAct Thought → Action → Observation 动态规划、高可解释性、动态纠错 需要外部工具、探索性任务
Plan-and-Solve 规划 → 执行 结构性强、稳定性好 逻辑路径确定、多步推理任务
Reflection 执行 → 反思 → 优化 显著提升解决方案质量 对准确性要求极高、代码生成
